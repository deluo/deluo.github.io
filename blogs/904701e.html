css两种盒模型

发表于 2018-08-04 | 分类于 web前端 | 本文总阅读量 次

W3C盒模型和IE盒模型

盒模型这个概念想必是每个学习css的初学者必然会接触到的.当然也是面试中常见问题.CSS处理网页时,它认为每个元素都包含在一个盒子里,这个盒子就是我们常说的盒模型 .这里主要介绍的是两种盒模型的区别:

W3C标准盒模型

css的width,height属性仅包括content内容部分

IE盒模型

css的width,height属性不仅包括content内容部分,还要包括padding和border

这里用一张图来进行对比就比较清楚了

两个盒模型实际显示效果的区别:

1.假如在布局中给一个盒子预留了200px宽度给它,设置width:200px.这个时候显示效果和预期一致.不论哪个盒模型都是没问题的.

2.但是后面产品经理觉得不好看,要加个边框,里面还要左右留白一部分.这时需要设置盒子的padding和border的值

3.重点来了,这个时候两种盒模型区别就体现出来了.

  • W3C盒模型 : 实际宽度 = content(width) + padding + border = 200px + 20px + 2px 已经大于了预期的200px宽度,width = content

  • IE盒模型 : 实际宽度 = content + padding + border = 178px + 20px + 2px 实际宽度等于预期200px宽度,

    width = content + padding + border

其实对于多数人来说,IE盒模型更加直观.如果要采用IE盒模型也很简单, CSS3 box-sizing属性

box-sizing

语法 box-sizing : content-box | border-box | inherit

content-box(默认值): 默认宽度和高度为内容content的宽高 等同于W3c盒模型

border-box: 宽高度为内容宽高和内边距加上边框之和 等同于IE盒模型

所以如果想按照IE盒模型那种计算宽高的方式,设置box-sizing属性为border-box即可
